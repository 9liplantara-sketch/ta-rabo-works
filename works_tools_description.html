Standalone reference

これまでに作成した各ツールの説明

本ページは、同一プロジェクト内にある他の HTML ページ（ポータルや各アプリ）へリンクせず、 ツールの目的と機能だけを文章でまとめた独立ドキュメントです。ブラウザで単体のファイルとして開いて読むことを想定しています。

01 — PROFILE

自己紹介インタラクティブ図解

制作者のプロフィール、哲学、活動領域、作品・受賞・活動の経緯、専門ドメインなどを、1 枚のスクロール可能な Web ページ上でインタラクティブに閲覧できる図解です。

画面構成は大きく分けて、ヒーロー（視覚的な導入）、哲学のカード、フォースレイアウトによる活動マップ（ノードグラフ）、タイムライン形式の Works、ドメイン別のカードなどに分かれており、静的 HTML・CSS・JavaScript のみで動作します。

閲覧者はスクロールと UI 操作を通じて情報の密度を自分のペースで調整でき、紙の履歴書やスライド一枚では伝えにくい関係性や時系列をまとめて把握しやすくなることを意図しています。

02 — LAB TOOL

研究室マネージャー

素材・スケジュール・学生の進捗・日報を1 つのアプリ画面の中で扱う、研究室運営向けの支援ツールです。サーバーを立てず、ブラウザ上のローカルストレージにデータを保存する設計です。

主な機能の概要

  • 素材ガイド：陶芸・漆・金属・木材など、実材に紐づく技法を整理する。素材ごとに複数の技法をタブで切り替え、手順の箇条書きや注意事項、技法ごとの簡易図（SVG など）を表示・追記できる。
  • カレンダー：複数メンバー（教員と学生など）の空き日を色分けして入力し、全員が重なる日を研究会などの候補日として扱える。長期タスクの登録にも対応する。
  • 学生進捗：マイルストーンのチェックリストと、研究メモ（テキストや画像）を学生単位で管理する。
  • 日報：日付・担当・活動内容・次のアクションを記録し、振り返り用のログとして残す。
  • ホーム（ダッシュボード）：各セクションの要約や、期日が近いタスクなどを一覧し、日々の確認の起点にする。

マテリアルや空間を扱う研究室において、安全・手順・スケジュール・記録を同じ UI 言語で扱えるようにすることが狙いです。

03 — FRAMEWORK

授業デザイン図解（授業設計フレームワーク）

授業やワークショップを設計するときの流れを、複数フェーズに分けたインタラクティブなフレームワークとして可視化したページです。キャンバス上のノード操作と、フェーズ別のタブ・詳細パネルを組み合わせ、設計の論点を順に埋めていけるようにしています。

フェーズは色分けされており、例として「目的・ゴール」「体験設計」「内容・素材」「流れ・構成」「評価・発展」といった軸で整理されます（実装では各フェーズに応じたラベルと入力欄が用意されている）。講師が 1 人で設計を練るときにも、チームで画面を共有しながら言語化を揃えるときにも使える想定です。

00 — PORTAL

ポータルトップ

上記ツール群への入口として、カード型レイアウトで各コンテンツを紹介するトップページがあります。本説明ページとは別ファイルであり、ここではリンクは張っていませんが、役割は「作品・ツール一覧の表紙」です。

META

Cursor 用エージェントスキル（プロフィール図解）

リポジトリ内には、IDE の AI エージェント向けにプロフィール図解 HTML の更新・再生成手順をまとめたスキル定義（Markdown）と、デザイントークン・アクティビティマップのデータ仕様・HTML スニペットなどの参照ファイルが含まれます。これはブラウザで開く図解そのものではなく、開発・保守を支援するためのメタ情報です。

技術面の共通事項（概要）
これらのページは原則としてバニラ HTML / CSS / JavaScript で構築され、アイコンは CDN 経由の Iconify、フォントは Google Fonts を利用しているものが多いです。データ永続化が必要なツールでは localStorage を用い、オフラインに近い環境でも単一ファイルまたは同一フォルダ内で動かしやすい構成にしています。
